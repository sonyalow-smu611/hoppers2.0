# Product Requirement Document (PRD)

## 1. App Overview

- **What is this app?**  
  Hoppers is a café discovery app that helps users quickly find cafés that match their preferences, location, budget, and current needs.

- **Core Value Proposition:**  
  Hoppers reduces the time and effort needed to choose a café by combining personalized recommendations, live map discovery, community reviews, real-time café conditions, promotions, and group decision-making in one platform.

## 2. Target Audience

- **Who is it for?**  
  Primary users are students, remote workers, café hoppers, friend groups, and casual diners looking for suitable cafés nearby.  
  Secondary users include café owners who want to promote deals, attract customers, and improve visibility.

- **User Pain Points:**  
  Users often struggle to quickly decide which café to visit because information is scattered across maps, social media, reviews, and promotion pages. They may not know whether a café is crowded, noisy, suitable for studying, budget-friendly, or aligned with their group’s preferences.

## 3. Core Features (MVP Scope)

- [ ] **Live Map Café Discovery:**  
  Users can view nearby cafés on a live map. The system recommends cafés based on user location and selected preferences such as budget, distance, purpose, crowd level, and noise level.

- [ ] **AI Café Recommender:**  
  Users can input preferences such as budget, purpose, distance, desired noise level, and desired crowd level. The AI recommender suggests the most suitable café options in the area and provides a brief summary explaining why each café fits.

- [ ] **Community Feed and Reviews:**  
  Users can upload café photos and write reviews for food, ambience, and value. Users may also import existing reviews from Google sources where available.

- [ ] **Personal Saved List:**  
  Users can save favourite cafés and organize them using tags such as “study spot,” “brunch spot,” “date spot,” or “work café.”

- [ ] **Real-Time Café Condition Feedback:**  
  Users at a café can submit quick feedback on crowd level, noise level, and temperature. The app displays summarized condition indicators so other users can decide if the café currently fits their needs.

- [ ] **Current Deals and Promotions:**  
  Users can browse cafés with active deals or promotions, helping them find wallet-friendly café options.

- [ ] **Group Decision Voting Room:**  
  Users can create a group voting room, invite friends, and vote on which café to visit. The app helps the group reach a decision based on votes and shared preferences.

## 4. Success Criteria & Metrics

- **Functional Success:**  
  Users can discover a suitable café within 3 main actions: set preferences, view recommendations, and select a café. Users can also save cafés, submit reviews, and create group voting rooms successfully.

- **Technical Performance:**  
  The app should load café recommendations and map results quickly, maintain stable performance during map browsing, and avoid unhandled runtime crashes during key flows such as recommendation, review upload, saving cafés, and voting.

- **Business/Adoption Goals:**  
  Users actively save cafés, submit condition feedback, and participate in group voting rooms. The platform should show meaningful engagement through repeated café searches, saved lists, review submissions, and promotion views.
